## Quick Setup Guide

Follow these steps to integrate AutoUI into your React + Vite + TypeScript project:

### Step 1: Install AutoUI

Install the AutoUI library from npm:

```bash
npm install @autoai-ui/autoui
```

**Peer Dependencies:**
- React >= 19.0.0 < 20.0.0
- React DOM >= 19.0.0 < 20.0.0

<Note title="Package Manager" type="info">
You can use `npm`, `pnpm`, or `yarn` - all are supported. Use whichever package manager your project uses.
</Note>

### Step 2: Get Your App ID and Shared Secret

Register your application on the [AutoUI Service Portal](https://autoui-chi.vercel.app/) to receive your credentials:

1. Visit [https://autoui-chi.vercel.app/](https://autoui-chi.vercel.app/)
2. Create a new app registration
3. You will receive:
   - **`appId`**: A unique identifier for your application (e.g., `app_1768313360453_dbptv83`)
   - **`shared_secret`**: A secret key for authenticating requests to the LLM proxy

<Note title="Service Portal" type="success">
The AutoUI Service Portal handles the backend proxy for you - no need to set up your own server! Simply register your app and use the provided credentials.
</Note>

### Step 3: Configure Environment Variables

Create a `.env` file in your project root (or `.env.local` for local development):

```bash
VITE_AUTOUI_APP_ID=your-app-id-here
VITE_BASE_URL=https://autoui-proxy.onrender.com
VITE_AUTOUI_SHARED_SECRET=your-shared-secret-here
```

<Note title="Environment Variables" type="warning">
Make sure to use the `VITE_` prefix for all environment variables so they're accessible in your Vite application. Never commit `.env` files with real secrets to version control - use `.env.local` for local development and add `.env.local` to `.gitignore`.
</Note>

### Step 4: Add the Vite Plugin

Add the AutoUI type schema plugin to your `vite.config.ts`:

```typescript
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
import { autouiTypeSchemaPlugin } from '@autoai-ui/autoui/plugin';

export default defineConfig({
  plugins: [
    react(),
    autouiTypeSchemaPlugin({
      // Optional: Override appId from env
      // appId: 'your-app-id',
      // Optional: Override version from env
      // version: '1.0.0',
    }),
  ],
});
```

<Note title="Plugin Purpose" type="info">
The Vite plugin automatically generates a `.autoui-runtime-schema.json` file during development builds. This schema file describes your app's capabilities to the LLM. Commit this file to your repository - it's used in production builds.
</Note>

### Step 5: Create Your AutoUI Configuration

Create a configuration file `src/lib/autoui-config.ts`:

```typescript
import type { AutoUIConfig } from '@autoai-ui/autoui';

export const autouiConfig: AutoUIConfig = {
  // Your app ID from the service portal
  appId: import.meta.env.VITE_AUTOUI_APP_ID || 'your-app-id',

  // LLM Configuration
  llm: {
    proxyUrl: import.meta.env.VITE_BASE_URL || 'https://autoui-chi.vercel.app',
    sharedSecret: import.meta.env.VITE_AUTOUI_SHARED_SECRET,
    appDescriptionPrompt: 'A brief description of what your app does',
    temperature: 0.7,
    maxTokens: 2048,
  },

  // Runtime Configuration
  runtime: {
    validateLLMOutput: true,
    storeChatToLocalStorage: true,
    localStorageKey: 'autoui_chat_history',
    enableDebugLogs: true, // Set to false in production
    maxSteps: 20,
    errorHandling: {
      showToUser: true,
      retryOnFail: false,
    },
  },

  // Function Definitions (start with an empty object, add functions as needed)
  functions: {},

  // Component Definitions (start with an empty object, add components as needed)
  components: {},

  // Optional Metadata
  metadata: {
    appName: 'Your App Name',
    appVersion: '1.0.0',
    description: 'Description of your application',
    tags: ['react', 'vite', 'autoui'],
  },
};
```

### Step 6: Integrate ModalChat

Import and use the `ModalChat` component in your main app file (e.g., `src/App.tsx`):

```tsx
import { ModalChat } from '@autoai-ui/autoui';
import { autouiConfig } from './lib/autoui-config';

function App() {
  return (
    <div className="App">
      <h1>My Application</h1>
      {/* Your app content */}
      
      {/* AutoUI Chat Modal */}
      <ModalChat config={autouiConfig} />
    </div>
  );
}

export default App;
```

### Step 7: Start Your Development Server

Start your development server:

```bash
npm run dev
```

You should now see a floating chat button in the bottom-right corner of your app! Click it to open the chat interface and start interacting with your AI assistant.

## Best Practices

### Configuration Management

<Note title="Centralize Your Config" type="success">
Keep your AutoUI configuration in a dedicated file (`src/lib/autoui-config.ts`). This makes it easier to maintain and update as your app grows.
</Note>

**Do:**
```typescript
// src/lib/autoui-config.ts
export const autouiConfig: AutoUIConfig = {
  // ... your config
};
```

**Don't:**
```typescript
// Don't define config inline in components
function App() {
  const config = { /* ... */ }; // Hard to maintain
  return <ModalChat config={config} />;
}
```

### Environment Variables

<Note title="Use Environment Variables" type="warning">
Always use environment variables for sensitive data like `appId` and `sharedSecret`. Never hardcode these values in your source code.
</Note>

**Do:**
```typescript
appId: import.meta.env.VITE_AUTOUI_APP_ID!,
sharedSecret: import.meta.env.VITE_AUTOUI_SHARED_SECRET,
```

**Don't:**
```typescript
appId: 'app_1768313360453_dbptv83', // Never hardcode!
sharedSecret: 'my-secret-key', // Never hardcode!
```

### Error Handling

<Note title="Enable Validation" type="success">
Always enable `validateLLMOutput` in production to catch errors early:
</Note>

```typescript
runtime: {
  validateLLMOutput: true, // Recommended for production
  errorHandling: {
    showToUser: true,
    retryOnFail: false, // Let the LLM retry, don't auto-retry
  },
}
```

### Debug Logging

<Note title="Debug Logs" type="info">
Enable debug logs during development, but disable them in production for better performance:
</Note>

```typescript
runtime: {
  enableDebugLogs: import.meta.env.DEV, // true in dev, false in prod
}
```

### Schema File

<Note title="Commit Schema File" type="success">
The `.autoui-runtime-schema.json` file generated by the Vite plugin should be committed to your repository. It's used in production builds.
</Note>

Add to `.gitignore` only if you want to regenerate it each time:
```
# Don't ignore - commit this file!
# .autoui-runtime-schema.json
```

## Best Ideas for Implementation

### 1. Start Simple, Add Complexity Gradually

<Note title="Incremental Approach" type="success">
Start with a minimal configuration and add functions and components as you need them. Don't try to register everything at once.
</Note>

**Phase 1: Basic Setup**
```typescript
functions: {},
components: {},
```

**Phase 2: Add Core Functions**
```typescript
functions: {
  getUserData: { /* ... */ },
  searchItems: { /* ... */ },
},
```

**Phase 3: Add UI Components**
```typescript
components: {
  ItemCard: { /* ... */ },
  ItemList: { /* ... */ },
},
```

### 2. Organize Functions by Domain

Group related functions together in your config:

```typescript
functions: {
  // User-related functions
  getUserProfile: { /* ... */ },
  updateUserSettings: { /* ... */ },
  
  // Data-related functions
  fetchProducts: { /* ... */ },
  searchProducts: { /* ... */ },
  
  // Navigation functions
  navigateToPage: { /* ... */ },
  goBack: { /* ... */ },
},
```

### 3. Use Descriptive Prompts

<Note title="Clear Prompts" type="success">
Write clear, descriptive prompts for your functions and components. The LLM uses these to understand when and how to use them.
</Note>

**Good:**
```typescript
fetchUserProfile: {
  prompt: 'Fetch the current user profile information including name, email, and avatar. Use this when the user asks about their profile or account information.',
  // ...
}
```

**Bad:**
```typescript
fetchUserProfile: {
  prompt: 'Gets user data', // Too vague!
  // ...
}
```

### 4. Provide Realistic Defaults

When registering components, provide realistic default props that match your data structure:

```typescript
components: {
  ProductCard: {
    prompt: 'Display a product card',
    defaults: {
      id: '1',
      name: 'Example Product',
      price: 29.99,
      image: 'https://example.com/product.jpg',
    },
    // ...
  },
}
```

### 5. Leverage Type Safety

Use TypeScript types for better development experience:

```typescript
import type { AutoUIConfig, AutoUIFunction } from '@autoai-ui/autoui';

interface Product {
  id: string;
  name: string;
  price: number;
}

const fetchProducts: AutoUIFunction = {
  prompt: 'Fetch products',
  callFunc: async (): Promise<Product[]> => {
    // TypeScript will help catch errors
    return products;
  },
  returns: 'Product[]',
};
```

### 6. Test Your Functions

Before registering functions, test them independently:

```typescript
// Test your function first
async function testFetchProducts() {
  const result = await fetchProducts();
  console.log('Products:', result);
}

// Then register it
functions: {
  fetchProducts: {
    prompt: 'Fetch products',
    callFunc: fetchProducts,
    returns: 'Product[]',
  },
}
```

### 7. Use Categories and Tags

Organize your components and functions with categories and tags:

```typescript
components: {
  ProductCard: {
    prompt: 'Display a product card',
    category: 'products',
    tags: ['display', 'products', 'card'],
    // ...
  },
}

functions: {
  searchProducts: {
    prompt: 'Search for products',
    tags: ['products', 'search', 'query'],
    // ...
  },
}
```

### 8. Monitor and Debug

Enable debug logs during development to understand how the LLM is using your functions:

```typescript
runtime: {
  enableDebugLogs: true,
}
```

Check the browser console for detailed logs about function calls and component renders.

## Common Patterns

### Pattern 1: Data Fetching

```typescript
functions: {
  fetchData: {
    prompt: 'Fetch data from API',
    params: {
      endpoint: 'string — API endpoint URL',
    },
    callFunc: async ({ endpoint }) => {
      const response = await fetch(endpoint);
      if (!response.ok) throw new Error('Failed to fetch');
      return response.json();
    },
    returns: 'any — JSON response',
  },
}
```

### Pattern 2: State Management

```typescript
// In your component
const [cart, setCart] = useState([]);

// In your config
functions: {
  addToCart: {
    prompt: 'Add item to cart',
    params: {
      itemId: 'string — item ID to add',
    },
    callFunc: ({ itemId }) => {
      setCart(prev => [...prev, itemId]);
      return { success: true, cartSize: cart.length + 1 };
    },
    returns: 'Object with success status',
  },
}
```

### Pattern 3: Navigation

```typescript
import { useNavigate } from 'react-router-dom';

const navigate = useNavigate();

functions: {
  navigateToPage: {
    prompt: 'Navigate to a page',
    params: {
      path: 'string — route path',
    },
    callFunc: ({ path }) => {
      navigate(path);
      return { success: true, currentPath: path };
    },
    returns: 'Navigation result',
  },
}
```

## Troubleshooting

If you encounter any issues:

1. **Check your environment variables** - Make sure they're prefixed with `VITE_`
2. **Verify your credentials** - Ensure `appId` and `sharedSecret` are correct
3. **Check the browser console** - Enable debug logs to see detailed error messages
4. **Review the schema file** - Check that `.autoui-runtime-schema.json` was generated
5. **See [Troubleshooting Guide](/docs/troubleshooting)** - Common issues and solutions

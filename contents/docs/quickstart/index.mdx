Welcome to AUTOUI! This page is the fastest path to a working integration in a **Vite + React + TypeScript** app.

## What you’ll build

- You’ll install the library from npm
- You’ll register an app in our hosted portal to get your **`appId`** and **`sharedSecret`**
- You’ll drop in `<ModalChat />`
- You’ll add the Vite plugin so AUTOUI can **validate function params and component props at runtime**

## Quickstart (6 steps)

<Step>
<StepItem title="1) Install from npm">

```bash
npm i @autoai-ui/autoui
```

<StepItem title="2) Register your app (hosted portal)">
Go to the **[AutoUI Service Portal](https://autoui-chi.vercel.app/)** and:

- Login / Sign up
- Create a **new app**
- Paste your **OpenRouter API key** when prompted
- Copy the generated credentials:
  - **`appId`**
  - **`sharedSecret`**

<StepItem title="3) Render ModalChat in your app">
In your React app (typically near the root), render `<ModalChat />`:

```tsx
import { ModalChat } from "@autoai-ui/autoui"
import { autouiConfig } from "./autoui.config"

export function App() {
  return (
    <main>
      {/* your app */}
      <ModalChat config={autouiConfig} />
    </main>
  )
}
```

<StepItem title="4) Add the Vite plugin (type validation)">
Add `autouiTypeSchemaPlugin()` to your `vite.config.ts`. This plugin generates the runtime schema used for validation.

```ts
import { defineConfig } from "vite"
import react from "@vitejs/plugin-react"
import { autouiTypeSchemaPlugin } from "@autoai-ui/autoui/plugin"

export default defineConfig({
  plugins: [react(), autouiTypeSchemaPlugin()],
})
```

By default the runtime schema is written to <code>.autoui-runtime-schema.json</code> (commit it to your repo so production builds can validate too).
</Note>

<StepItem title="5) Copy/paste the config template (and register functions/components)">
Create an env file (example: <code>.env.local</code>) and fill in your values:

```bash
# From the portal:
#
# IMPORTANT:
# - `AUTOUI_APP_ID` is read by the Vite plugin
# - `VITE_AUTOUI_APP_ID` is used by your frontend code
# They MUST be the same value.
AUTOUI_APP_ID=app_xxx
VITE_AUTOUI_APP_ID=app_xxx
VITE_AUTOUI_SHARED_SECRET=xxx

# Default hosted proxy (recommended):
VITE_AUTOUI_PROXY_URL=https://autoui-proxy.onrender.com

# If you're hosting your own backend proxy, use your URL instead:
# VITE_AUTOUI_PROXY_URL=https://your-proxy.example.com
```

Now create a config file (example: <code>src/autoui.config.ts</code>) and paste this template:

```ts
import type { AutoUIConfig } from "@autoai-ui/autoui"

// ✅ Import your actual functions/components (replace with your real paths)
import MyWidget from "./components/MyWidget"
import { doSomething } from "./lib/doSomething"

export const autouiConfig: AutoUIConfig = {
  /* =========================
   *   APP ID (IMPORTANT)
   * =========================
   * Your actual appId from the AutoUI Service Portal
   */
  appId: import.meta.env.VITE_AUTOUI_APP_ID,

  llm: {
    /**
     * Default hosted proxy:
     * - https://autoui-proxy.onrender.com
     * If self-hosting the backend, put your own proxy URL in the env instead.
     */
    proxyUrl: import.meta.env.VITE_AUTOUI_PROXY_URL,

    /**
     * Your actual sharedSecret from the AutoUI Service Portal
     * (This is NOT your OpenRouter API key.)
     */
    sharedSecret: import.meta.env.VITE_AUTOUI_SHARED_SECRET,

    appDescriptionPrompt: "Describe your app in 1-2 sentences.",
    temperature: 0.2,
    maxTokens: 2048,
  },

  runtime: {
    validateLLMOutput: true,
    storeChatToLocalStorage: true,
    localStorageKey: "autoui_chat",
    enableDebugLogs: true,
    maxSteps: 20,
    errorHandling: {
      showToUser: true,
      retryOnFail: false,
    },
  },

  /* =========================
   *   FUNCTIONS
   * =========================
   * Put your actual app functions here (the assistant will be able to call them)
   */
  functions: {
    /* your actual functions here */
    doSomething: {
      prompt: "Explain what this function does and when the assistant should call it.",
      callFunc: doSomething, // ✅ your actual function here
      // canShareDataWithLLM: true,
    },
  },

  /* =========================
   *   COMPONENTS
   * =========================
   * Put your actual React components here (the assistant will be able to render them)
   */
  components: {
    /* your actual components here */
    MyWidget: {
      prompt: "Explain what UI this component renders and when it’s useful.",
      callComponent: MyWidget, // ✅ your actual component here
      defaults: {
        /* your default props here (recommended) */
      },
      exampleUsage: "<MyWidget />",
      category: "layout",
      // tags: ["dashboard"],
    },
  },
}
```

The final step is to **register** the same functions/components so the plugin can extract their types (and runtime validation can work):

```tsx
import {
  autouiRegisterComponentPropsSchema,
  autouiRegisterFunctionParamsSchema,
} from "@autoai-ui/autoui"

import MyWidget from "./components/MyWidget"
import { doSomething } from "./lib/doSomething"

autouiRegisterComponentPropsSchema(MyWidget)

autouiRegisterFunctionParamsSchema(doSomething)
```

<Note title="Important" type="info">
You typically do both:

- Put functions/components into <code>config.functions</code> and <code>config.components</code> (so AUTOUI can call/render them)
- Register them (so the Vite plugin can generate schema and runtime validation can work)
</Note>

Want a deeper explanation of what each prop does and how to structure this cleanly?  
See: **[Config functions & components guide](/docs/config/functions-and-components)**

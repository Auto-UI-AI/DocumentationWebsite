Complete reference for the `AutoUIConfig` type.

## Type Definition

```tsx
import type { AutoUIConfig } from "@autoai-ui/autoui"
```

## Top-Level Fields

### `appId` (required)

**Type:** `string`

Unique identifier for your application.

```tsx
appId: 'tasks-demo'
```

Used for:
- Logging and analytics
- localStorage keys (if not overridden)
- Error tracking

### `metadata` (optional)

**Type:** `AutoUIMetadata`

Application metadata that describes your app to the LLM. This is optional but recommended for better LLM understanding.

```tsx
metadata: {
  appName: string
  appVersion?: string
  author?: string
  createdAt?: string // ISO timestamp
  description?: string // Detailed description for LLM
  tags?: string[]
}
```

**Fields:**
- `appName`: Display name of your application (required)
- `appVersion`: Version string (e.g., "1.0.0") (optional)
- `author`: Author or team name (optional)
- `createdAt`: ISO 8601 timestamp (optional)
- `description`: Detailed description that tells the LLM what your app does (optional but recommended)
- `tags`: Array of tags for categorization (optional)

### `llm` (required)

**Type:** `LLMConfig`

LLM provider configuration.

```tsx
llm: {
  proxyUrl: string                         // Required: Backend proxy URL
  sharedSecret?: string                    // Optional: Shared secret for proxy auth
  temperature?: number                     // Optional: Sampling temperature hint (0.0-2.0)
  maxTokens?: number                       // Optional: Max tokens hint
  appDescriptionPrompt?: string            // Optional: High-level app context
  requestHeaders?: Record<string, string>  // Optional: Headers forwarded to proxy
}
```

**Required fields:**
- `proxyUrl`: URL of your proxy server (REQUIRED in production)

**Optional fields:**
- `sharedSecret`: Client key generated by the portal/proxy (optional, for proxy auth)
- `temperature`: Randomness hint (0.0-2.0)
- `maxTokens`: Maximum response length hint
- `appDescriptionPrompt`: Concise description of app capabilities
- `requestHeaders`: Optional headers forwarded to proxy

### `runtime` (required)

**Type:** `RuntimeConfig`

Runtime execution configuration.

```tsx
runtime: {
  validateLLMOutput?: boolean
  storeChatToLocalStorage?: boolean
  localStorageKey?: string
  enableDebugLogs?: boolean
  maxSteps?: number
  errorHandling?: {
    showToUser?: boolean
    retryOnFail?: boolean
  }
  runtimeSchemaPath?: string
}
```

**Fields:**
- `validateLLMOutput`: Validate plan JSON structure (recommended: `true`, optional)
- `storeChatToLocalStorage`: Persist chat history (optional)
- `localStorageKey`: Key for localStorage (default: `'autoui_chat'`, optional)
- `enableDebugLogs`: Log execution details to console (optional)
- `maxSteps`: Maximum instruction steps allowed (optional)
- `errorHandling.showToUser`: Display errors in chat UI (optional)
- `errorHandling.retryOnFail`: Auto-retry failed function calls (optional)
- `runtimeSchemaPath`: Path (or URL) to the runtime schema file. Default: `'.autoui-runtime-schema.json'` (optional)

### `functions` (required, but can be empty)

**Type:** `Record<string, AutoUIFunction>`

Registered functions that the assistant can call.

```tsx
functions: {
  functionName: {
    prompt: string                                    // Required: Description for LLM
    params?: Record<string, string>                  // Optional: Parameter descriptions
    callFunc: Function                               // Required: The actual function implementation
    returns?: string                                 // Optional: Return type description
    exampleUsage?: string                           // Optional: Example usage for LLM
    tags?: string[]                                 // Optional: Tags for organization
    canShareDataWithLLM?: boolean                   // Optional: Whether function can share data with LLM
  }
}
```

**Important:** Functions must return a value (not void) so the LLM can analyze the results.

See [Functions](/docs/functions) for detailed documentation.

### `components` (required, but can be empty)

**Type:** `Record<string, AutoUIComponent>`

Registered React components for generative UI.

```tsx
components: {
  componentName: {
    prompt: string                                  // Required: Description for LLM
    props?: Record<string, string>                  // Optional: Human-readable prop descriptions
    callComponent: ComponentType<any>              // Required: React component reference
    defaults?: Record<string, any>                   // Optional: Default prop values
    exampleUsage?: string                           // Optional: Example JSX usage (string)
    category?: string                               // Optional: Category (e.g., 'product-display', 'checkout')
    tags?: string[]                                 // Optional: Tags for search or grouping
  }
}
```

**Note:** `callComponent` must be a React `ComponentType`, which can be a function component or class component.

See [Components](/docs/components) for detailed documentation.

## Complete Example

```tsx
const config: AutoUIConfig = {
  appId: 'my-app',
  
  // Optional but recommended
  metadata: {
    appName: 'My Application',
    appVersion: '1.0.0',
    author: 'Your Name',
    createdAt: new Date().toISOString(),
    description: 'Detailed description of what your app does...',
    tags: ['demo', 'react'],
  },
  
  llm: {
    proxyUrl: 'https://autoui-proxy.onrender.com',  // Required
    sharedSecret: 'your-shared-secret',             // Optional
    temperature: 0.2,                   // Optional
    maxTokens: 2048,                   // Optional
    appDescriptionPrompt: 'A simple application.', // Optional
    requestHeaders: {                   // Optional
      'HTTP-Referer': 'https://myapp.com',
    },
  },
  
  runtime: {
    validateLLMOutput: true,           // Optional
    storeChatToLocalStorage: true,      // Optional
    localStorageKey: 'autoui_chat',     // Optional
    enableDebugLogs: false,             // Optional
    maxSteps: 20,                      // Optional
    errorHandling: {                    // Optional
      showToUser: true,
      retryOnFail: false,
    },
  },
  
  functions: {
    // Optional
    myFunction: {
      prompt: 'Does something useful',
      params: {
        param1: 'string - Description of param1',
      },
      callFunc: ({ param1 }) => {
        // Implementation
        return { result: 'success' }
      },
      returns: 'Object with result',
      canShareDataWithLLM: true,
    },
  },
  
  components: {
    // Optional
    MyComponent: {
      prompt: 'Displays something useful',
      props: {
        title: 'string - Component title',
      },
      callComponent: ({ title }) => <div>{title}</div>,
      defaults: {
        title: 'Default Title',
      },
      category: 'display',
    },
  },
}
```

## TypeScript Types

If you're using TypeScript, import the types:

```tsx
import type { AutoUIConfig, LLMConfig, RuntimeConfig, AutoUIMetadata } from "@autoai-ui/autoui"
```

## Validation

AUTOUI validates your config at runtime. Common validation errors:

- Missing required fields (`appId`, `llm.proxyUrl`)
- Invalid `proxyUrl` format
- Invalid `maxSteps` (must be positive number if provided)
- Invalid `temperature` (must be 0.0-2.0 if provided)
- Functions that return `void` (functions must return a value)

## Best Practices

- Use descriptive `appId` values
- Write detailed `metadata.description` for better LLM understanding
- Keep `llm.appDescriptionPrompt` concise but informative
- Enable `validateLLMOutput` in production
- Use meaningful `localStorageKey` values
- Set appropriate `maxSteps` based on app complexity

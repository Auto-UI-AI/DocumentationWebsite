Complete reference for the `AutoUIConfig` type.

## Type Definition

```tsx
import { AutoUIConfig } from "@autoai-ui/autoui"
```

## Top-Level Fields

### `appId` (required)

**Type:** `string`

Unique identifier for your application.

```tsx
appId: 'tasks-demo'
```

Used for:
- Logging and analytics
- localStorage keys (if not overridden)
- Error tracking

### `metadata` (required)

**Type:** `Metadata`

Application metadata that describes your app to the LLM.

```tsx
metadata: {
  appName: string
  appVersion: string
  author: string
  createdAt: string // ISO timestamp
  description: string // Detailed description for LLM
  tags: string[]
}
```

**Fields:**
- `appName`: Display name of your application
- `appVersion`: Version string (e.g., "1.0.0")
- `author`: Author or team name
- `createdAt`: ISO 8601 timestamp
- `description`: **Critical** - Detailed description that tells the LLM what your app does
- `tags`: Array of tags for categorization

### `llm` (required)

**Type:** `LLMConfig`

LLM provider configuration.

```tsx
llm: {
  proxyUrl: string
  sharedSecret: string
  appDescriptionPrompt: string
  temperature?: number // Default: 0.2
  maxTokens?: number   // Default: 2048
}
```

**Required fields:**
- `proxyUrl`: URL of your proxy server
- `sharedSecret`: Client key generated by proxy server
- `appDescriptionPrompt`: Concise description of app capabilities

**Optional fields:**
- `temperature`: Randomness (0.0-2.0), default: 0.2
- `maxTokens`: Maximum response length, default: 2048

### `runtime` (required)

**Type:** `RuntimeConfig`

Runtime execution configuration.

```tsx
runtime: {
  validateLLMOutput: boolean
  storeChatToLocalStorage: boolean
  localStorageKey?: string
  enableDebugLogs: boolean
  maxSteps: number
  errorHandling: {
    showToUser: boolean
    retryOnFail: boolean
  }
}
```

**Fields:**
- `validateLLMOutput`: Validate plan JSON structure (recommended: `true`)
- `storeChatToLocalStorage`: Persist chat history
- `localStorageKey`: Key for localStorage (default: `'autoui_chat'`)
- `enableDebugLogs`: Log execution details to console
- `maxSteps`: Maximum steps per plan execution
- `errorHandling.showToUser`: Display errors in chat UI
- `errorHandling.retryOnFail`: Auto-retry failed function calls

### `functions` (optional)

**Type:** `Record<string, FunctionConfig>`

Registered functions that the assistant can call.

```tsx
functions: {
  functionName: {
    prompt: string
    params?: Record<string, string>
    callFunc: (params: any) => any
    returns?: string
  }
}
```

See [Functions](/docs/functions) for detailed documentation.

### `components` (optional)

**Type:** `Record<string, ComponentConfig>`

Registered React components for generative UI.

```tsx
components: {
  componentName: {
    prompt: string
    defaults?: Record<string, any>
    callComponent: (props: any) => React.ReactElement
    category?: string
  }
}
```

See [Components](/docs/components) for detailed documentation.

## Complete Example

```tsx
const config: AutoUIConfig = {
  appId: 'my-app',
  
  metadata: {
    appName: 'My Application',
    appVersion: '1.0.0',
    author: 'Your Name',
    createdAt: new Date().toISOString(),
    description: 'Detailed description of what your app does...',
    tags: ['demo', 'react'],
  },
  
  llm: {
    proxyUrl: 'http://localhost:3001',
    sharedSecret: 'your-client-key',
    appDescriptionPrompt: 'A simple application.',
    temperature: 0.2,
    maxTokens: 2048,
  },
  
  runtime: {
    validateLLMOutput: true,
    storeChatToLocalStorage: true,
    localStorageKey: 'autoui_chat',
    enableDebugLogs: false,
    maxSteps: 20,
    errorHandling: {
      showToUser: true,
      retryOnFail: false,
    },
  },
  
  functions: {
    // Optional
  },
  
  components: {
    // Optional
  },
}
```

## TypeScript Types

If you're using TypeScript, import the types:

```tsx
import { AutoUIConfig, Metadata, LLMConfig, RuntimeConfig } from "@autoai-ui/autoui"
```

## Validation

AUTOUI validates your config at runtime. Common validation errors:

- Missing required fields (`appId`, `metadata`, `llm`, `runtime`)
- Invalid `proxyUrl` format
- Missing `sharedSecret`
- Invalid `maxSteps` (must be positive number)
- Invalid `temperature` (must be 0.0-2.0)

## Best Practices

- Use descriptive `appId` values
- Write detailed `metadata.description` for better LLM understanding
- Keep `llm.appDescriptionPrompt` concise but informative
- Enable `validateLLMOutput` in production
- Use meaningful `localStorageKey` values
- Set appropriate `maxSteps` based on app complexity
